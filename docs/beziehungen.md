# 🔗 Beziehungen — Wer arbeitet wo?

Beziehungen verbinden **Kontakte** mit **Unternehmen**. Sie bilden ab, in welcher Rolle eine Person bei einer Organisation tätig ist — und das kann sich über die Zeit ändern.

---

## So funktionieren Beziehungen

Eine Beziehung verknüpft immer eine Person mit einem Unternehmen und beschreibt:

| Feld | Beschreibung |
|------|-------------|
| **Kontakt** | Die verknüpfte Person |
| **Unternehmen** | Die verknüpfte Organisation |
| **Beziehungstyp** | Die Art der Verbindung (Angestellter, Geschäftsführer, etc.) |
| **Position** | Die konkrete Jobtitel/Funktion |
| **Beginn** | Seit wann besteht die Beziehung? |
| **Ende** | Bis wann? (leer = aktuell aktiv) |
| **Hauptansprechpartner** | Ist diese Person der primäre Kontakt für das Unternehmen? |
| **Notizen** | Zusätzliche Informationen |

---

## Beziehungstypen

Das System kennt viele gängige Rollen:

| Typ | Typische Verwendung |
|-----|-------------------|
| 👔 Geschäftsführer | CEO, Managing Director |
| 📋 Prokurist | Handlungsbevollmächtigter |
| 👥 Abteilungsleiter | Head of Department |
| 💼 Manager | Teamleiter, Projektmanager |
| 👤 Angestellter | Regulärer Mitarbeiter |
| 🎯 Berater | Externer Consultant |
| 💻 Freelancer | Freier Mitarbeiter |
| 📦 Lieferant | Zulieferer-Kontakt |
| 🤝 Kunde | Kundenseitiger Ansprechpartner |
| 🤝 Partner | Geschäftspartner |
| 💰 Investor | Kapitalgeber |
| 🏛️ Aufsichtsrat | Kontrollgremium |
| 📊 Beirat | Beratendes Gremium |

---

## Mehrere Rollen gleichzeitig

Eine Person kann **gleichzeitig mehrere Beziehungen** haben:

- Max Müller ist **Geschäftsführer** bei Firma A
- Max Müller ist **Beirat** bei Firma B
- Max Müller ist **Investor** bei Firma C

Alle drei Verbindungen existieren parallel. So bildet das CRM die reale Welt ab, in der Menschen oft in mehreren Kontexten aktiv sind.

---

## Beziehungen über die Zeit

Beziehungen haben ein **Start-** und **Enddatum**. Wenn jemand das Unternehmen verlässt, setzt du das Enddatum — die Beziehung bleibt als historischer Eintrag erhalten.

> 💡 **Warum löschen wir nicht?** Weil die Geschichte wertvoll ist. Wenn du weißt, dass ein Kontakt früher bei einem bestimmten Unternehmen war, kann das für künftige Gespräche relevant sein.

---

## Hauptansprechpartner

Pro Unternehmen kannst du einen Kontakt als **Hauptansprechpartner** markieren. Das ist die Person, die du als erstes kontaktierst, wenn du mit dem Unternehmen kommunizieren willst.

---

## Beziehungen verwalten

Du kannst Beziehungen von beiden Seiten aus pflegen:

- Auf der **Kontaktseite** siehst du alle Unternehmen, mit denen die Person verbunden ist
- Auf der **Unternehmensseite** siehst du alle Personen, die dort eine Rolle haben

Neue Beziehungen erstellst du auf beiden Seiten — das Ergebnis ist dasselbe.
